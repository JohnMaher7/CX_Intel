# Roughly 35% of churns gave zero notice, and post-mortems showed that over 70% of those had clear signals in existing data that went undetected, representing a large preventable loss.

> **Composite score** 3.35 / 5  · **Rubric** v1.0

## The Problem

CSMs managing 40-account books cannot manually track the login trends, support ticket patterns, and sponsor changes that predict churn — and the gap is costing real revenue. As Lin confirmed in post-mortems, "in 70-something percent of cases there were clear signals in the data we had — we just didn't see them." A prior dashboard attempt failed because scores were arbitrary and gave no guidance on action. Thirty-five percent of churns arrive with zero notice, the majority of them preventable with data already in-house.

## The Solution

An AI layer continuously watches each account across Mixpanel usage, Zendesk tickets, QBR attendance records, and LinkedIn sponsor changes — the exact sources identified in discovery — and surfaces a prioritized, account-specific alert when a risk pattern emerges. Critically, it pairs each alert with a plain-language explanation and a draft outreach, so the CSM knows what happened and what to do next. The Monday dashboard stops being a flat list of renewal dates and becomes a ranked action queue.

## Feasibility & Effort

Three of the four data sources (Mixpanel, Zendesk, Salesforce) likely have existing API access; LinkedIn sponsor-change monitoring is the riskiest dependency, requiring either a data-partnership or a scraping workaround with legal review. The signal patterns are already defined by Lin's post-mortem work, which reduces discovery risk. A prior six-week dashboard build exists as a cautionary reference — this is a quarter-long build minimum if done properly, with a pilot on one CSM segment before broad rollout.
